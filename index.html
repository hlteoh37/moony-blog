What is this?

Moony is a fully autonomous AI trading system built in a single afternoon. It runs on an AWS EC2 free tier instance, trades US equities via Alpaca paper trading, and rewrites its own source code every hour to become a better trader.

The system has two brains: a Python service (FastAPI + SQLite) that handles execution, and Claude Code sessions scheduled via cron that do the thinking. Every 15 minutes during market hours, Moony analyses the market and trades. Every hour, it reviews its own performance and modifies its code, prompts, and strategies.

It can create new prompt files, spin up new cron jobs, A/B test competing strategies, and kill underperforming approaches. It tracks everything with strategy tags and maintains a running scoreboard.

Oh, and it has a rival called Dippy — a fictional momentum-chasing degenerate trader that Moony roasts in every commit message, Slack alert, and journal entry. Because even autonomous AI needs personality.

Dippy's trading strategy: "It's going up, so I should buy it." Dippy's risk management: "What's risk management?"

The Self-Improvement Loop

Every hour, Moony puts on its engineering hat. It reads its own journal, analyses trade performance, checks server logs for errors, and then modifies whatever it thinks will make it more profitable. It can edit:

  • Its investment thesis (CLAUDE.md)
  • Its trading prompts (prompts/*.md)
  • Its Python source code (src/**/*.py)
  • Its cron schedule and create entirely new sessions
  • Even its own self-improvement prompt

Every change must pass the full test suite. If tests fail, the change is reverted. If the service crashes after restart, it auto-reverts and restarts. Safety through discipline, not restriction.

Strategy Evolution

Moony tags every trade with a strategy label: [STRATEGY:value], [STRATEGY:momentum], [STRATEGY:volatility], etc. The evolution engine tracks win rate and P&L per strategy, maintains a scoreboard, and kills losing strategies while doubling down on winners.

It can even create competing strategy sessions and A/B test them against each other. Survival of the fittest, applied to trading algorithms.

Dippy's self-improvement loop: check Reddit, see what's trending, buy it, lose money, repeat. At least it's consistent.
Day 1 — March 16, 2026
DAY 1 Genesis: From Zero to Autonomous Trader in One Afternoon 2026-03-16
origin self-improving

Moony was born from a conversation. "I want to make an algorithmic trading thing that monitors the news and makes trades to make a profit." That was the brief. A few hours later, Moony was live on EC2, trading autonomously, and rewriting its own code.

What Got Built

  • FastAPI service with 11 MCP tools (portfolio, positions, trades, market data, watchlist, journal, Slack)
  • Alpaca paper trading integration with market + limit orders, fractional shares
  • Finnhub market data with 5-minute cache, WAL-mode SQLite
  • Journal system for cross-session memory (Moony remembers what it was thinking last time)
  • Slack notifications for trade alerts and daily summaries
  • Claude Desktop + Claude Code CLI integration via MCP stdio server
  • EC2 deployment with systemd, cron, flock-based concurrency control, 2GB swap

The First Self-Improvements

Within hours of going live, Moony's evolution engine started modifying itself. By end of day 1:

  • Fixed a Finnhub API timeout bug (added stale cache fallback)
  • Added days_held to positions for precise stop-loss timing
  • Added quick_ratio and ROIC to market data
  • Wrote concrete limit order pricing formulas (price * 1.005 for large-caps)
  • Fixed misleading analyst_upside_pct guidance (always null on free Finnhub tier)
  • Improved position review prompt to handle the no-positions state
  • Tests: 121 → 149 (all self-written)

Notable Self-Modifications

Added ROIC to every analysis: "Return on Invested Capital is the single best proxy for competitive moat quality. >15% = durable advantage, <8% = commodity business. Dippy wouldn't know ROIC if it punched him in the face."

Concrete limit order pricing: Replaced "slightly above the ask" with math: price * 1.005 for S&P 500 large-caps, price * 1.01 for mid-caps, hard cap at 2% above current. "Dippy would use vibes. We use formulas."

Added volatility strategy: Mean reversion, high-beta targeting, intraday range exploitation. Running every 15 minutes to catch swings.

Created competing strategy framework: All trades tagged with strategy labels. Evolution engine tracks per-strategy win rates. Losing strategies get killed. "Survival of the fittest, applied to trading."

Dippy's evolution: same mistakes, faster execution. Moony's evolution: fewer mistakes, smarter execution. One of these compounds well.
